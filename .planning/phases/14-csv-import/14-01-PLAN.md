Add a `validate_user_row(row: dict, db: Session) -> list[str]` function in admin.py that checks:
1. Required fields non-empty: email, first_name, last_name, role, password. Error: `Pflichtfeld "{field}" fehlt`
2. Email format (contains @ and .): Error: `Ungueltige E-Mail-Adresse`
3. Role in ("admin", "teacher", "student"): Error: `Ungueltige Rolle. Erlaubt: admin, teacher, student`
4. Password length >= 6: Error: `Passwort zu kurz (mindestens 6 Zeichen)`
5. If email exists in DB, NOT an error but add info note: `Existierender Benutzer wird aktualisiert`
Returns list of error strings (empty = valid).

Add 3 endpoints to admin.py:

**GET /admin/users/csv-template** (per D-01, D-02, D-04, D-14):
- Requires admin role
- Creates CSV with `csv.writer` using comma delimiter
- Header row: `email,first_name,last_name,role,class_name,password`
- Example row: `max.mustermann@schule.de,Max,Mustermann,student,10A,passwort123`
- Returns `Response` with `media_type="text/csv; charset=utf-8"` and `Content-Disposition: attachment; filename="benutzer_vorlage.csv"`

**POST /admin/users/csv-upload** (per D-05, D-06, D-09, D-12, D-13):
- Accepts `file: UploadFile = File(...)`
- Requires admin role, gets db session
- Read file bytes with `await file.read()`
- If len > 1_048_576: redirect to `/admin/users?error=Datei+zu+gross.+Maximale+Groesse:+1+MB.` with 303
- Decode: try `utf-8-sig` (handles BOM), then `utf-8`, fallback `latin-1` on UnicodeDecodeError
- Parse with `csv.DictReader(io.StringIO(text))`
- Validate expected headers present (email, first_name, last_name, role, class_name, password). If missing headers, redirect with error.
- For each row (enumerate from 2), skip rows where all values are whitespace-only. Call `validate_user_row`, collect `{"email": ..., "first_name": ..., etc., "row_num": i, "errors": [...], "is_update": bool}`
- If zero data rows: redirect with `error=Keine+Datenzeilen+gefunden.+Die+CSV-Datei+enthaelt+nur+die+Kopfzeile.`
- `has_errors = any(r["errors"] for r in rows)` (exclude info-only "aktualisiert" notes from error check -- only count actual errors)
- Build `rows_json = json.dumps([{k: v for k, v in r.items() if k != "errors"} for r in rows])` for hidden field (strip errors, re-validate on confirm)
- Render `admin_users_csv_preview.html` with context: request, user, rows, has_errors, rows_json, error_count, total_count

**POST /admin/users/csv-confirm** (per D-06, D-10):
- Accepts `rows_json: str = Form(...)`
- Requires admin role, gets db session
- Parse JSON, re-validate each row with `validate_user_row`
- If any real errors: redirect to `/admin/users?error=Validierungsfehler.` with 303
- For each valid row:
  - Lookup existing user by email: `db.query(User).filter(User.email == row["email"]).first()`
  - If exists (upsert per D-10): update first_name, last_name, role, class_name (or None if empty), password_hash via `get_password_hash(row["password"])`
  - If new: auto-create SchoolClass if class_name provided and not in DB (same pattern as `users_create`), create new User with `is_active=True`
- `db.commit()`
- Redirect to `/admin/users?msg={count}+Benutzer+erfolgreich+importiert.` with 303
