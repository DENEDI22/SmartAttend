# Phase 4 — UI Design Contract

> Visual and interaction contract for the Teacher Interface. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Pico CSS 2.x (classless, served from `/static/pico.min.css`) |
| Icon library | none (Unicode symbols only: checkmark, dash, bullet) |
| Font | Pico CSS default system font stack |

**Note:** This project uses Pico CSS classless styling with server-rendered Jinja2 templates. No JS framework, no build step, no component registry. All styling is achieved through semantic HTML elements and Pico CSS custom properties.

---

## Spacing Scale

Pico CSS provides its own spacing via semantic elements. Inline style overrides use these values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline button padding (vertical), compact gaps |
| sm | 8px | Gap between inline action buttons, compact padding |
| md | 16px | Grid gap between form fields, default element spacing |
| lg | 24px | Section padding inside `<article>` elements |
| xl | 32px | `margin-top` between major page sections (established in admin templates as `2rem`) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: none

---

## Typography

Pico CSS controls all typography through its default styles. The following documents what Pico applies to the semantic elements used in this phase:

| Role | Element | Size | Weight | Line Height |
|------|---------|------|--------|-------------|
| Body | `<p>`, `<td>` | 16px (1rem) | 400 (regular) | 1.5 |
| Label / Small | `<small>`, `<th>` | 14px (0.875rem) | 600 (semibold for `<th>`) | 1.5 |
| Heading | `<h1>` | 32px (2rem) | 700 (bold) | 1.2 |
| Subheading | `<h2>` via `<hgroup>` | 16px (1rem) | 400 (regular) | 1.5 |

**Constraint:** Do NOT override Pico's font sizes or weights with inline styles. Use semantic elements (`<h1>`, `<small>`, `<strong>`) to achieve visual hierarchy.

---

## Color

All colors use Pico CSS custom properties to maintain light/dark mode compatibility. Direct hex values are only used where Pico has no semantic variable.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--pico-background-color)` | Page background, table background |
| Secondary (30%) | `var(--pico-card-background-color)` | `<article>` cards (empty state, summary header) |
| Accent (10%) | `var(--pico-primary)` | Primary CTA links ("Details" links in dashboard table), `<a>` elements |
| Success | `#2e7d32` | Success flash messages, attendance status checkmark color |
| Muted | `var(--pico-muted-color)` | Absent student status dash, secondary text |
| Destructive | `var(--pico-del-color)` | Error flash messages only |

Accent reserved for: "Details" links in the dashboard table, "CSV herunterladen" link in lesson detail, navigation links in `<nav>`.

**Title:** `Stundenplan heute -- SmartAttend`

| Element | HTML Pattern | Notes |
|---------|-------------|-------|

**Template:** `teacher_lesson.html` extends `teacher_base.html`
**Title:** `Anwesenheit {Klasse} -- SmartAttend`

| Element | HTML Pattern | Notes |
|---------|-------------|-------|
| Summary header | `<hgroup><h1>Anwesenheit {Klasse}</h1><p>{Raum} -- {start_time} bis {end_time} -- Anwesend: {X}/{Y}</p></hgroup>` | Full context in subheading |
| Roster table | `<figure><table>...</table></figure>` | Full class roster, not just checked-in |
| Table columns | `Nachname / Vorname / Status / Uhrzeit` | 4 columns |
| Present status | `<td style="color: #2e7d32">Anwesend</td>` | Green checkmark text |
| Absent status | `<td style="color: var(--pico-muted-color)">Abwesend</td>` | Muted dash text |
| Check-in time | `<td>08:15</td>` or `<td>--</td>` | Format `HH:MM`, empty for absent |
| CSV link | `<a href="/teacher/lesson/{token_id}/csv" role="button" class="outline">CSV herunterladen</a>` | Below the table, styled as outline button |
| Back link | `<a href="/teacher">&larr; Zurueck zur Uebersicht</a>` | Above the heading |

### Endpoint: CSV Export (`/teacher/lesson/<token_id>/csv`)

No template -- returns `text/csv` response directly.

| Property | Value |
|----------|-------|
| Content-Type | `text/csv; charset=utf-8` |
| Content-Disposition | `attachment; filename="Anwesenheit_{Klasse}_{YYYY-MM-DD}.csv"` |
| Encoding | UTF-8 with BOM (`\xef\xbb\xbf`) |
| Separator | Semicolon (`;`) |
| Columns | `Nachname;Vorname;Klasse;Status;Uhrzeit` |
| Present row | `Mueller;Max;10A;Anwesend;08:15` |
| Absent row | `Schmidt;Anna;10A;Abwesend;` |

## Interaction Contracts

### Dashboard Auto-Refresh
- No auto-refresh. Teacher manually reloads the page (per REQUIREMENTS.md: "Real-time attendance updates (WebSocket) -- Not needed; page refresh is sufficient").
- No JavaScript required for this phase.

### Lesson Roster Sorting
- Students sorted alphabetically by `last_name` ascending, then `first_name` ascending.
- Sorting is server-side, not client-side.

### Token Not Yet Issued
- When a ScheduleEntry exists for today but no AttendanceToken has been created yet (scheduler has not run):
  - Dashboard shows `--` in the Anwesend column.
  - No "Details" link is rendered for that row.
  - Tooltip or `<small>` note: none (keep it simple, `--` is self-explanatory).

### Access Control
- All `/teacher/*` routes require `require_role("teacher")`.
- Lesson detail view validates that `token.schedule_entry.teacher_id == current_user.id`. If mismatch, redirect to `/teacher?error=Sie+haben+keinen+Zugriff+auf+diese+Stunde.`

### CSV Download
- Triggers browser download (Content-Disposition: attachment).
- No confirmation dialog needed (non-destructive action).
