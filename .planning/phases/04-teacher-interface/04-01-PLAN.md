    - For each lesson, count expected students: `User` where `role == "student"` AND `class_name == lesson.class_name` AND `is_active == True`
    - For each lesson, if token exists count AttendanceRecord where `token_id == token.id`, else checked_in = 0
    - Build a list of dicts with keys: `class_name`, `room`, `start_time` (formatted HH:MM), `end_time` (formatted HH:MM), `expected`, `checked_in`, `token_id` (or None if no token)
    - Pass to template: `request`, `lessons`, `weekday_name` (from WEEKDAY_NAMES), `today_str` (formatted DD.MM.YYYY), `active_page="overview"`

    **IMPORTANT: No "Fach" column.** ScheduleEntry has NO subject field (Phase 1 decision D-07). CONTEXT.md D-01 is authoritative and lists: Klasse, Raum, Zeit, Anwesend, Details-Link. The UI-SPEC mentions "Fach" speculatively -- ignore it.
