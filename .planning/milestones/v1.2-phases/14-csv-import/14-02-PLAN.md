  <action>
Add a `validate_schedule_row(row: dict, row_num: int, db: Session, previous_rows: list[dict]) -> list[str]` function in admin.py:

1. Required fields non-empty: device_id, teacher_email, class_name, weekday, start_time, end_time. Error: `Pflichtfeld "{field}" fehlt`
2. Device lookup: `db.query(Device).filter(Device.device_id == row["device_id"]).first()`. If not found: `Geraet "{device_id}" nicht gefunden`
3. Teacher lookup: `db.query(User).filter(User.email == row["teacher_email"], User.role == "teacher").first()`. If not found: `Lehrer mit E-Mail "{email}" nicht gefunden`
4. Weekday: try `int(row["weekday"])`, must be 0-6. Error: `Ungueltige Wochentag-Nummer (0-6)`
5. Time format: parse start_time and end_time as `datetime.strptime(val, "%H:%M").time()`. Error: `Ungueltiges Zeitformat (erwartet HH:MM)`
6. start_time < end_time. Error: `Startzeit muss vor Endzeit liegen`
7. If late_threshold_minutes provided and non-empty: must be a positive integer. Error: `Ungueltige Verspaetungsminuten`
8. DB overlap: call `check_schedule_conflict(db, device.id, weekday, start_time, end_time)`. Error: `Ueberschneidung mit bestehendem Eintrag`
9. Intra-CSV overlap: for each previous_row that has same device_id and weekday, check if time ranges overlap (s1 < e2 and s2 < e1). Error: `Ueberschneidung mit Zeile {N} in dieser Datei`

Returns list of error strings. Also returns resolved device.id and teacher.id (use a tuple return or store in row dict).

Add 3 endpoints:

**GET /admin/schedule/csv-template** (per D-01, D-03, D-04):
- Requires admin role
- Header: `device_id,teacher_email,class_name,weekday,start_time,end_time,late_threshold_minutes`
- Example: `e101,lehrer@schule.de,10A,0,08:00,09:30,`
- Returns Response with `media_type="text/csv; charset=utf-8"`, `Content-Disposition: attachment; filename="stundenplan_vorlage.csv"`

**POST /admin/schedule/csv-upload** (per D-05, D-07, D-09, D-11, D-12, D-13):
- Same file reading/decoding pattern as user CSV (1MB limit, BOM detection, latin-1 fallback)
- Validate expected headers: device_id, teacher_email, class_name, weekday, start_time, end_time (late_threshold_minutes optional)
- For each row: call `validate_schedule_row` passing `previous_rows` (list of already-validated rows for intra-CSV overlap check)
- Build rows with: all CSV fields + row_num, errors, resolved device_id (int), resolved teacher_id (int)
- Build rows_json for hidden field: include device_id string, teacher_email, class_name, weekday, start_time, end_time, late_threshold_minutes
- Render `admin_schedule_csv_preview.html`

**POST /admin/schedule/csv-confirm**:
- Parse rows_json, re-validate each row (same validation + overlap check)
- If errors: redirect to `/admin/devices?error=Validierungsfehler.`
- For each valid row: resolve device and teacher, auto-create SchoolClass if needed, create `ScheduleEntry(device_id=device.id, teacher_id=teacher.id, class_name=row["class_name"], weekday=int(row["weekday"]), start_time=parsed_start, end_time=parsed_end, late_threshold_minutes=int(val) if val else None)`
- `db.commit()`
- Redirect to `/admin/devices?msg={count}+Stundenplaneintraege+erfolgreich+importiert.`
